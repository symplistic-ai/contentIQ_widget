# ContentIQ Chat Widget

A lightweight, customizable chat widget for embedding ContentIQ AI assistants on your website.

## Usage

### Recommended: iframe embed (Cloudflare Pages)

Generated by the ContentIQ Deploy UI (`loader.js` on `contentiq-widget.pages.dev`):

```html
<script defer src="https://contentiq-widget.pages.dev/loader.js"
  data-agent="YOUR_AGENT_ID"
  data-token="YOUR_SITE_TOKEN"
  data-backend="https://backend.contentiq.symplistic.ai"
  data-sso="true"></script>
```

The loader injects a cross-origin iframe (`embed.html`) so widget code and SSO state are isolated from your page scripts.

### Via jsDelivr CDN (legacy inline)

```html
<div class="contentiq_symplisticai_chat" data-agent="YOUR_AGENT_ID"></div>
<script src="https://cdn.jsdelivr.net/gh/symplistic-ai/contentIQ_widget@prod/dist/widget.min.js"
  data-agent="YOUR_AGENT_ID"
  data-token="YOUR_SITE_TOKEN"
  data-backend="https://backend.contentiq.symplistic.ai"></script>
```

### Legacy iframe (deprecated)

`dist/widget.html` with query parameters is superseded by `loader.js` + `embed.html`. Prefer the loader snippet for new deployments.

## Deployed assets (Cloudflare Pages `dist/`)

| File | Purpose |
|------|---------|
| `loader.js` | Customer-page bootstrap |
| `embed.html` | Iframe shell |
| `widget.js` | Widget runtime |
| `widget.min.js` | Minified runtime (jsDelivr) |
| `sso_callback.html` | Entra SSO callback (popup + redirect) |
| `_headers` | CSP / cache for Pages |

After changing `widget.js`, regenerate `widget.min.js`:

```bash
npx terser dist/widget.js -c -m -o dist/widget.min.js
```

## Development

- `main` — development
- `prod` — stable for jsDelivr `@prod`

Source of truth for widget runtime: `contentIQ_frontend/widget.js` in the main ContentIQ monorepo. Sync into `dist/` before release.
